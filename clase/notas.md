## 🧠 Actividades realizadas en clase
Durante la sesión se trabajó en la integración de las vistas arquitectónicas utilizando el caso base FarmApp (cadena de farmacias con e-commerce) como referencia inicial.

- Se discutió con el equipo el concepto de integración de vistas y la idea de que una arquitectura empresarial no es una colección de diagramas aislados, sino un conjunto de vistas complementarias que deben ser coherentes entre sí.
- Se identificaron las cinco vistas a integrar para el caso FarmApp: negocio (procesos de compra, prescripción y despacho), información (entidades Producto, Cliente, Pedido, Descuento), aplicaciones (app móvil, plataforma e-commerce, POS, CRM), infraestructura (servidores regionales, nube híbrida, base de datos replicada) y seguridad (control de accesos por rol, cifrado de datos personales, monitoreo de fraude).
- Se tomó la decisión de organizar el tablero en cuatro bandas horizontales apiladas (negocio → información → aplicaciones → infraestructura) y representar la seguridad como una banda transversal, dado que las amenazas atraviesan todas las capas y no constituyen un nivel más.
- Se analizó cómo se conectan las capas entre sí, definiendo conexiones verticales (hilos conductores) que permiten rastrear un proceso de negocio hasta la infraestructura que lo soporta.
- Se utilizó la herramienta draw.io para estructurar el tablero de vistas integradas del caso base FarmApp.
- Se recibió retroalimentación del docente sobre la coherencia entre capas y la claridad de las conexiones.
- Se logró completar el tablero del caso base FarmApp y definir la estrategia para aplicar la misma integración al cliente real CEMEDICA IPS.
